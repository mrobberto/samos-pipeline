# SAMOS Pipeline## OverviewThis repository contains the working reduction pipeline for SAMOS spectroscopy,from raw detector frames through:- frame orientation- bias correction- cosmic-ray cleaning- row-stripe removal- trace construction- pixel-flat correction- science rectification- wavelength calibration- 1D extraction- OH cleanup- telluric correction- photometric flux calibrationThe pipeline is designed to be:- **config-driven**- **step-based**- **re-runnable**- **inspectable through QC products**- **usable both from scripts and notebooks**The current operational science flow is:- **Step09** = ABAB OH-clean stage- **Step10** = telluric correction- **Step11** = flux calibration---## Repository structure```textsamos-pipeline/├── calibration/        # reference calibration files and tables├── config/             # generic pipeline config + target-specific profiles├── drivers/            # pipeline orchestration layer├── pipeline/           # production step scripts├── qc/                 # quality-control scripts└── ...config/Configuration is split into three layers:pipeline_config.pygeneric pipeline helperscanonical product filenamesstage directory conventionsreductions/run8_dolidze25.pyactive target-specific reduction profiletarget/night rootsfile selectionscanonical full paths for productstarget_config.pythin selector for the currently active target profiledrivers/The main operational entry point is:PYTHONPATH=. python drivers/run_pipeline.pyThis driver:runs the pipeline in canonical stage ordersupports partial execution (--from-step, --to-step, --only)supports parity selection (--set EVEN|ODD|ALL)optionally runs QC (--run-qc)validates canonical outputs after key stagesNotebooks for interactive execution and review are stored in:drivers/Jupyter/pipeline/This folder contains the production step scripts.Each step lives in its own folder, for example:pipeline/step04_traces/pipeline/step07_wavecal/pipeline/step09_oh_refine/pipeline/step10_telluric/pipeline/step11_fluxcal/Only the active production scripts remain in these folders. Older prototypes andsuperseded variants have been moved into pipeline/archive/.qc/This folder contains the active quality-control scripts used for:intermediate validationper-step reviewfinal diagnostic summariesnotebook displaysdriver-integrated QC runsOnly the current QC layer remains in the active step folders. Older orsuperseded QC scripts have been moved into qc/archive/.Current stage modelThe current operational stages are:00 orientation01 master bias02 bias correction03 cosmic-ray cleaning03.5 row-stripe cleanup04 trace construction05 pixel flat06a final science mosaic06b apply pixel flat06c rectify to TRACECOORDS07a–07h wavelength calibration chain08a–08c 1D extraction chain09 ABAB drivers/run_pipeline.py --from-step 11a --to-step 11cRun with QCPYTHONPATH=. python drivers/run_pipeline.py --from-step 09 --to-step 11c --run-qcNotebooksThe notebooks under drivers/Jupyter/ are the interactive front end for:step-by-step executionvisual inspectiondevelopment/debuggingscience validationThey complement the pipeline but do not replace the canonical driver.Canonical philosophyProduction logic lives in:pipeline/config/Orchestration lives in:drivers/run_pipeline.pyInspection and validation live in:qc/notebooksArchived material lives in:pipeline/archive/qc/archive/Archived files are retained for provenance and recovery, but they are notthe operational path.Maintenance notesWhen updating the repository:update config/pipeline_config.py for generic naming conventionsupdate the active reduction profile in config/reductions/update drivers/run_pipeline.py if stage structure changesupdate active QC scripts only when they are part of the operational workflowmove superseded material into archive/ rather than leaving it mixed into theactive foldersThe goal is to keep the active repository layer readable and unambiguous.StatusThe repository is currently organized around a clean, config-driven reductionflow with:one canonical driverone active target profileone operational script path per stepone active QC layer per steparchived historical material preserved separately---# `ARCHIVE.md` at repo root```markdown# Archive PolicyThis repository retains older scripts,
